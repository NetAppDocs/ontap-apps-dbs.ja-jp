= VMwareデータストア向けFabricPool
:hardbreaks:
:allow-uri-read:

:icons: font

[role="lead"]
仮想マシンのストレージはゲスト OS とハイパーバイザー層から ONTAP へと流れ、そこで FabricPool 階層化と WAFL がパフォーマンス、効率性、コスト最適化を実現します。

ONTAP 9.6リリース以降、FabricPoolはVMwareデータストアを含む仮想化ワークロードをサポートしています。VMwareデータストア（VMテンプレート、電源オフ状態のVM、スナップショット、バックアップなど）をバックアップするFlexVolボリュームからコールドブロックをオブジェクトストレージに階層化しつつ、ホットデータはローカルに保持できるようになりました。

とはいえ、注意すべき細かな点がいくつかあります。

== コンピューティング層と仮想ディスク層

ゲスト オペレーティング システムは、VMDK、QCOW2、VHDX などの仮想ディスク フォーマット上で動作し、アプリケーションやユーザーがアクセスできるコンピューティング層とブロック デバイス層を作成します。

ゲスト オペレーティング システムは仮想ディスクをブロック デバイスとして認識するため、無停止アップグレード、ハードウェア障害、その他の予期しないイベント発生時の耐障害性を計画する際には、ゲスト内のブロック デバイスのタイムアウトを考慮する必要があることに注意してください。


TIP: link:https://www.netapp.com/pdf.html?item=/media/17239-tr-4598.pdf["FabricPoolのベストプラクティス"]に従って、ゲストOSに関連する仮想ディスクファイルのブロックの性質により、NASプロトコルを使用している場合でも、仮想マシンのデータストアをホストするFlexVolボリュームをSANストレージとして扱う必要があります。したがって、NetAppではsnapshotのみの階層化ポリシーの使用を推奨します。autoポリシーは重要度の低いデータに使用できますが、クラウド階層との接続が切断されると、マシンがクラッシュしたり、読み取り専用状態になったりする可能性があるため注意してください。



== ハイパーバイザーの役割

ハイパーバイザーは、実行中の仮想マシンと、CPU、メモリ、ストレージなどの物理リソースとの間の仲介役を務めます。NASのユースケースでは、ハイパーバイザーがNAS共有をマウントし、仮想ディスクファイルをブロックデバイスとして仮想マシンに提示します。

ハイパーバイザー ファイル システム パスには、iSCSI/FCP接続のLUNまたはVMFS、NTFS、EXT4などのハイパーバイザー固有のファイル システムでフォーマットされたNVMeネームスペースを介したVMストレージが含まれます。

NAS ファイル システム パスは、NFS または SMB プロトコルを使用して、仮想マシン ストレージ用の ONTAP ボリュームへの直接ファイルレベル アクセスを提供します。

両方のパスは ONTAP の WAFL レイヤーで収束し、スナップショットやメタデータ管理などの一貫したデータ サービスを実現します。



== FabricPool階層化統合

FabricPoolは、ハイパフォーマンスなローカルストレージと低コストのクラウドオブジェクト階層の間でデータ配置をインテリジェントに管理します。

FabricPoolは、データセットに適用されたFabricPool階層化ポリシーに基づいて、ローカルのハイパフォーマンス ストレージとスケーラブルなクラウド オブジェクト ストレージの間でデータを自動的に階層化します。

ONTAPのFabricPool統合により、ローカル読み取りのパフォーマンスを犠牲にすることなく、ハイブリッドクラウド環境全体で効率的なデータ配置が可能になります。



== ストレージの効率性と管理

FabricPoolを有効にする前に、その仕組みを理解することが重要です。適切な構成を行うことで、適切なデータを適切な階層に保存し、パフォーマンスの向上、リスクの低減、TCOの最適化を実現できます。

SAN/NVMeとNASファイル プロトコル パスは、どちらもWAFL層に集約されます。WAFLファイル システムは、メタデータの整合性を確保し、Snapshotとクローニングをサポートし、SSD、HDD、オブジェクト ストレージ メディアへの書き込みを最適化します。
