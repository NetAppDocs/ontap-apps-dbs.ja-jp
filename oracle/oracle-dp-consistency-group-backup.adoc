= 整合グループのバックアップ
:allow-uri-read: 


[role="lead"]
最もシンプルなバックアップ方法としては、Oracleデータベース全体を単一のONTAPボリューム、LUN、またはNVMeネームスペースに配置するだけです。

データファイル、アーカイブログ、リドゥログ、および制御ファイルを単一のスナップショットで保護することは、有効なバックアップ、復元、およびレプリケーション方法です。しかし、RPOはバックアップ自体の時点に限定されます。RPOが1時間以上の場合に適しています。データベースがAFFボリュームまたはASAストレージユニットにまたがる場合、整合性グループの設定には、前述のツールのいずれかを使用する必要があります。

例えば、データベース全体を単一のAFFボリュームに配置し、以下のSnapshotスケジュールを設定できます。

* 72時間ごとのスナップショット
* 30個の夜間Snapshot
* 12の月次スナップショット


これにより、過去72時間のローリング期間において1時間のRPO（目標復旧時点）が実現され、さらに毎晩および毎月のバックアップも実行されます。複数のデータベースやアプリケーション ファイルを単一のボリュームまたは整合性グループのスナップショット セットに含めることで、より大規模な環境全体で一貫性のあるバックアップを実現することもできます。
