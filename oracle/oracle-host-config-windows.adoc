= Microsoft Windows の場合
:allow-uri-read: 


[role="lead"]
ONTAPを使用したMicrosoft Windows上のOracleデータベースの構成に関するトピック



== NFS

Oracleは、direct NFSクライアントでのMicrosoft Windowsの使用をサポートしています。この機能により、NFSの管理上のメリット（環境間でのファイルの表示、NFSボリュームの動的なサイズ変更、より低コストなIPプロトコルの活用など）を実現できます。DNFSを使用したMicrosoft Windows上でのデータベースのインストールと設定については、Oracleの公式ドキュメントを参照してください。特別なベストプラクティスはありません。



== SAN

圧縮効率を最適化するには、NTFSファイルシステムで8K以上の割り当て単位を使用するようにしてください。一般にデフォルトである4Kの割り当て単位を使用すると、圧縮効率が低下します。
